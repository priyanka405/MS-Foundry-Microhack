Background

The Problem Worth Solving

Every company has the same productivity tax: employees spend time hunting for answers that already exist somewhere — but are hard to reach.

The Daily Friction

HR portals are buried, IT tickets take days, policy documents are out-of-date PDFs. Support teams are flooded with repetitive questions while employees wait.

An AI agent connected to the right knowledge and tools can resolve most of these queries in seconds — and this MicroHack shows you exactly how to build one correctly, using Microsoft Foundry.

  • Policies and procedures are scattered across systems
  • Repetitive support tasks consume HR and IT bandwidth
  • Employees expect instant, accurate, contextual answers
  • Enterprise requirements add safety and compliance constraints

Employees Are Asking…

How do I request vacation days?
How do I reset my password?
Where is the travel policy document?
Who approves my expense report?
Can you open an IT ticket for my broken laptop?
Use Case

What the Agent Does

The Employee Support Agent combines language understanding, knowledge retrieval, tool execution, and safety guardrails into a single, production-grade assistant.

💬

Answer Questions

Respond to employee queries in natural language — HR policies, IT procedures, benefits information, and more.

🔍

Search Documents

Query a grounded enterprise knowledge base (HR + IT documents) to provide accurate, cited answers instead of hallucinated responses.

🎫

Create IT Tickets

Call a mock ticketing API on behalf of the employee — no form hunting, no manual copy-paste, just conversational action.

🧠

Remember Preferences

Store per-user context (department, role, preferences) across sessions so the agent personalises every interaction.

🛡️

Follow Policies

Enforce company guardrails — restrict confidential data, prevent prompt injection, and comply with content-safety rules.

📊

Observable & Measurable

Every request is traced, evaluated for correctness, and monitored in Application Insights so you can operate it with confidence.

⚙️

Challenge 0 — Setup

Deploy a Foundry Project, Model, Storage, Monitoring, and optional Search using Bicep IaC templates.

🏷️

Challenge 1 — Model Choice

Benchmark GPT-4o vs. a small model on support Q&A tasks. Compare cost, latency, and answer quality.

🤖

Challenge 2 — First Agent

Create an agent with system instructions for the employee support scenario. Understand how agents differ from raw completions.

📖

Challenge 3 — Grounding

Add Foundry IQ with HR and IT knowledge-base documents. Fix hallucinations and make the agent reliable.

🔧

Challenge 4 — Tools

Give the agent the ability to create IT tickets through an MCP Tool or Foundry Toolbox. Turn assistant into agent.

💾

Challenge 5 — Memory

Implement session and user memory so the agent remembers an employee's department and preferences across conversations.

🔒

Challenge 6 — Guardrails

Add Content Safety, restrict confidential policy access, and protect against prompt injection attacks.

📊

Challenge 7 — Evaluation

Build a synthetic HR Q&A dataset, run evaluations, and score the agent on groundedness and correctness.

⚡

Challenge 8 — Optimization

Use evaluation results to improve system prompts and tool descriptions. Build → Evaluate → Optimize → Repeat.

🔭

Challenge 9 — Observability

Configure distributed traces, monitoring dashboards, and Application Insights. If you can't observe it, you can't operate it.

🏢

Challenge 10 — Publish

Publish the agent to your enterprise. Apply governance, security controls, and lifecycle management for production readiness.

Repository

Repository Structure

The repo follows the standard MicroHack layout — one folder per challenge, shared infrastructure, and separate guides for coaches and students.

MS-Foundry-Microhack/ ├── README.md ├── docs/ # GitHub Pages landing page │ ├── index.html │ └── styles.css ├── challenges/ │ ├── challenge0-setup/ │ ├── challenge1-models/ │ ├── challenge2-agent/ │ ├── challenge3-grounding/ │ ├── challenge4-tools/ │ ├── challenge5-memory/ │ ├── challenge6-guardrails/ │ ├── challenge7-evals/ │ ├── challenge8-optimization/ │ ├── challenge9-observability/ │ └── challenge10-publish/ ├── infra/ # Bicep templates │ ├── main.bicep │ ├── modules/ │ └── parameters/ ├── starter-code/ ├── coach-guide/ ├── student-guide/ └── solution-guide/

What Lives Where

  • docs/ — This GitHub Pages landing page. Enable Pages → Deploy from /docs.
  • challenges/ — One folder per challenge with instructions, starter code, and solution hints.
  • infra/ — Bicep modules for deploying all Azure Foundry resources (Challenge 0).
  • starter-code/ — Boilerplate Python/TypeScript to kickstart each challenge.
  • coach-guide/ — Facilitator notes, timing, and hints for each challenge.
  • student-guide/ — Step-by-step walkthroughs for participants.
  • solution-guide/ — Reference implementations for each challenge.
Get Started

Ready to Build?

Everything you need to run the MicroHack is in this repository. Follow the steps below to get your environment ready.

  1. Fork or clone the repository

    git clone https://github.com/priyanka405/MS-Foundry-Microhack.git
    Each participant needs their own copy.

  2. Complete Challenge 0 — Environment Setup

    Open challenges/challenge0-setup/ and follow the instructions to deploy your Azure Foundry environment using the Bicep templates in infra/.

  3. Work through each challenge in order

    Each challenge folder contains a README.md with the challenge brief, learning objectives, and hints. Use starter-code/ as your base.

  4. Check your work against the solution guide

    Reference implementations are in solution-guide/. Use them after completing each challenge, not before!

  5. Enable GitHub Pages to share your progress

    Go to Settings → Pages → Deploy from branch → /docs to publish this landing page at your own URL.
