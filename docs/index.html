Introduction

What is Microsoft Foundry?

Microsoft Foundry is a platform for building, evaluating, monitoring, and operating AI agents and agentic workflows across real enterprise scenarios.

Why Foundry matters

Foundry brings together models, orchestration, tools, knowledge grounding, evaluations, tracing, and deployment patterns into one developer workflow.

Instead of stopping at a demo chatbot, this MicroHack shows how to build a production-oriented agent that can answer questions, search enterprise content, call tools, remember context, and operate with guardrails.

  • Build agents with instructions, tools, and memory
  • Ground answers using enterprise documents and knowledge bases
  • Evaluate quality before shipping to users
  • Trace behavior and monitor production usage
  • Publish agents with enterprise governance in mind

Challenge

Challenge 1 — Model Choice

After setup, compare model options for support-style Q&A so you can balance quality, cost, and latency.

Why start here?

Model selection shapes the rest of the agent lifecycle. A support agent must answer accurately, quickly, and at a sustainable cost. This challenge helps participants understand those tradeoffs early.

  • Compare stronger and smaller models on the same task
  • Measure response quality for enterprise support scenarios
  • Discuss latency and cost tradeoffs before building more features
🧪

Expected outcome

Participants leave this challenge with a justified model choice for the Employee Support Agent, not just a default selection.
