Hackathon overview

Turn contract chaos into an AI-native workflow

Legal and Procurement burn 35% of their week on "what does clause X say?" and lose millions per year on missed auto-renewals. This MicroHack builds the assistant that ends that.

Create

Draft NDAs, MSAs, and SOWs from approved templates and clauses — with citations.

Search & compare

Retrieval over the enterprise corpus, side-by-side clause comparisons, and File Search on uploaded PDFs.

Automate

Route approvals via Logic Apps, generate documents, and track lifecycle state through renewal.

Prove quality

Prompt Shields + Content Safety + a Foundry-evaluated gate before any deploy.

Learning objectives

By the end, you can…

  1. Stand up a Foundry project with a model, indexed corpus, and enabled tracing.
  2. Build a grounded agent with a strong persona and refusal behavior.
  3. Ground the agent with Azure AI Search + File Search — with citations.
  4. Extend the agent with 5 tools (Search, File Search, Logic Apps, Functions, Code Interpreter).
  5. Protect the agent with Prompt Shields, PII detection, and app-layer blocklists.
  6. Trace every prompt, retrieval, tool call, and response into App Insights.
  7. Evaluate on a fixed dataset and enforce a deployment gate.
  8. Optimize model, prompt, retrieval, and cost with a repeatable loop.
  9. Publish as a Web App, a Teams app, or an authenticated API endpoint.
Challenge timeline

Nine progressive challenges

Do them in order — each one wires up the piece the next one needs.

  1. Setup

    Challenge 0 · Setup

    Create the Foundry project, deploy the model, connect Azure AI Search, enable tracing, and upload the corpus.

    • Foundry portal
    • Azure CLI
    • Blob upload
  2. Build

    Challenge 1 · Build the Contract Intake & Drafting Agent

    Write the persona and instructions, prove the first round-trip, and lock in the refusal behavior.

    • Agent Builder
    • Python SDK
    • gpt-4o
  3. Ground

    Challenge 2 · Knowledge Grounding

    Index NDAs, MSAs, policies, and the clause library into Azure AI Search. Add citations to every answer.

    • Azure AI Search
    • Blob
    • File Search
  4. Tools

    Challenge 3 · Tools & Actions

    Five tools: AI Search, File Search, Logic Apps (approval), Azure Functions (status/risk), Code Interpreter (reports).

    • Function calling
    • Logic Apps
    • Functions
  5. Protect

    Challenge 4 · Guardrails

    Prompt Shields, PII detection, approved-template enforcement, restricted clause modification.

    • Content Safety
    • Prompt Shields
    • Blocklists
  6. Observe

    Challenge 5 · Observability

    Trace every prompt → LLM → retrieval → tool → response into Application Insights. Write the KQL.

    • OpenTelemetry
    • App Insights
    • KQL
  7. Evaluate

    Challenge 6 · Evaluation

    Score groundedness, relevance, task adherence, safety, tool accuracy on a 15-row dataset. Turn the gate green.

    • Evaluators
    • Scorecards
    • CI gate
  8. Optimize

    Challenge 7 · Optimization

    Sweep model, prompt, retrieval, and tools. Show the before/after and the cost delta.

    • Model swap
    • Retrieval tuning
    • Cost
  9. Ship

    Challenge 8 · Publish

    Ship as a Web App with Easy Auth, a Teams app, or an authenticated API endpoint.

    • Web App
    • Teams
    • API
Foundry services used

Every service earns its place

Microsoft Foundry

Agent Service, model deployment, evaluators, tracing.

Azure OpenAI

gpt-4o and gpt-4o-mini for chat and function calling.

Azure AI Search

Vector + semantic hybrid over your contract corpus with citations.

Azure Blob Storage

Source-of-truth corpus feeding the Search index.

Azure Functions

Backing services for clause lookup, status, and risk scoring.

Logic Apps

Office 365 approval routing for Legal and Procurement.

Code Interpreter

On-the-fly summaries, obligation extraction, renewal reports.

Content Safety + Prompt Shields

Guardrails on inputs and outputs at the project level.

Application Insights

Cost, latency, and grounding telemetry with reusable KQL.

Two paths, one outcome

Low-code & pro-code — both fully documented

Low-code · Foundry portal

  • Point-and-click Agent Builder for persona and tools.
  • Portal-driven Azure AI Search connection.
  • Content Safety and Prompt Shields toggles.
  • In-portal Evaluations for the deployment gate.
  • One-click publish to a Foundry-hosted chat UI.

Pro-code · Foundry SDK (Python)

  • azure-ai-projects + azure-ai-agents for programmatic control.
  • azure-search-documents for the index and hybrid search.
  • azure-ai-evaluation for scriptable gates in CI.
  • azure-monitor-opentelemetry + @traced spans.
  • Reference implementation in app/; run python -m app.sample_run --smoke.
