Solution architecture

Every challenge builds one slice of this picture

Two views — what the agent does for the business, and which Azure services power it.

Functional architecture

Legal, Procurement, and Sales work with one Contract Lifecycle Management Agent that exposes four business capabilities:

  1. Intake & drafting — turn a request into a first draft from approved templates.
  2. Contract search & review — find contracts, quote clauses, and pull templates via Foundry IQ.
  3. Market & regulatory research — enrich answers with external context via WebIQ.
  4. Contract status & renewals — report lifecycle stage from Azure SQL; never miss a renewal.

The agent never gives legal advice.

Technical architecture

  • Channel — Web App (Easy Auth) / Teams / API.
  • Agent runtime — Azure AI Foundry (Agents + Models).
  • Grounding — Azure AI Search + Azure Blob Storage.
  • Business systems — Foundry IQ (Azure AI Search + SharePoint), WebIQ (Bing Search), Azure SQL (contract status).
  • Safety — Content Safety + Prompt Shields + PII detection.
  • Observability — Application Insights via OpenTelemetry.
  • CI gate — GitHub Actions + Azure AI Evaluation SDK.
Agent tools

Three tools the agent uses

Every user request is answered by orchestrating one or more of these. Every call is traced and evaluated.

Foundry IQ

Azure AI Search + SharePoint — contract search, document grounding, and knowledge retrieval across the corpus, templates, executed contracts, and policies.

Outcome: grounded answers with citations to the exact clause, paragraph, or SharePoint document.

WebIQ

Bing Search — external research and market intelligence: counterparty background, regulatory context, industry benchmarks.

Outcome: fresh, cited web sources augmenting the internal corpus.

Azure SQL

Azure SQL — structured contract metadata: status, dates, owners, renewal information, KPIs.

Outcome: deterministic status answers; renewals never missed.

Design decisions

Foundry IQ collapses AI Search + SharePoint into one internal-knowledge tool · WebIQ handles anything not in the corpus · Azure SQL is a direct data connector for structured contract state · safety wraps every tool.

Learning objectives

By the end, you can…

  1. Stand up a Foundry project with a model, indexed corpus, and enabled tracing.
  2. Build a grounded agent with a strong persona and refusal behavior.
  3. Ground the agent with Azure AI Search + File Search — with citations.
  4. Extend the agent with 3 tools: Foundry IQ (Azure AI Search + SharePoint), WebIQ (Bing Search), and Azure SQL (structured contract data).
  5. Protect the agent with Prompt Shields, PII detection, and app-layer blocklists.
  6. Trace every prompt, retrieval, tool call, and response into App Insights.
  7. Evaluate on a fixed dataset and enforce a deployment gate.
  8. Optimize model, prompt, retrieval, and cost with a repeatable loop.
  9. Publish as a Web App, a Teams app, or an authenticated API endpoint.

Foundry services used

Every service earns its place

Microsoft Foundry

Agent Service, model deployment, evaluators, tracing.

Azure AI Foundry Models

gpt-4o and gpt-4o-mini for reasoning, drafting, and language tasks.

Foundry IQ

One knowledge tool wrapping Azure AI Search (corpus, hybrid vector + semantic) and SharePoint (templates, executed contracts, policies).

Azure AI Search

Vector + semantic hybrid retrieval over the contract corpus with citations.

Azure Blob Storage

Source-of-truth corpus feeding the Search index.

SharePoint

Enterprise contract repository: approved templates, executed contracts, and policies.

WebIQ (Bing Search)

External research and market intelligence: counterparty, regulatory, benchmarks — cited to source URLs.

Azure SQL

System of record for contract lifecycle state: status, owner, renewal date, expiry, KPIs.

Content Safety + Prompt Shields

Guardrails on inputs and outputs at the project level.

Application Insights

Cost, latency, and grounding telemetry with reusable KQL.

Two paths, one outcome

Low-code & pro-code — both fully documented

Low-code · Foundry portal

  • Point-and-click Agent Builder for persona and tools.
  • Portal-driven Azure AI Search connection.
  • Content Safety and Prompt Shields toggles.
  • In-portal Evaluations for the deployment gate.
  • One-click publish to a Foundry-hosted chat UI.

Pro-code · Foundry SDK (Python)

  • azure-ai-projects + azure-ai-agents for programmatic control.
  • azure-search-documents for the index and hybrid search.
  • azure-ai-evaluation for scriptable gates in CI.
  • azure-monitor-opentelemetry + @traced spans.
  • Reference implementation in app/; run python -m app.sample_run --smoke.
