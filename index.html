Hackathon overview

An AI teammate for the whole contract lifecycle

Enterprise contract portfolios are large, unstructured, and expensive to reason about. This MicroHack shows you how to give Legal, Procurement, and Sales a real AI teammate — one that searches contracts, retrieves clauses, identifies risks, routes approvals, and tracks status across the lifecycle.

🔎

Contract search

Find any contract by counterparty, type, or free text — with citations.

📄

Clause retrieval

Termination, liability, indemnity, GDPR, payment — quoted verbatim.

⚖️

Contract comparison

Compare terms across agreements in a structured, side-by-side table.

📝

Contract summarization

Obligations, key dates, and risk flags in a clean contract brief.

🚨

Risk identification

Non-standard or missing clauses surfaced with a numeric risk score.

🚦

Approval routing

Route changes to Legal or Procurement via Logic Apps — with confirmation.

🏗️

Document generation

Draft NDAs, SOWs, and amendments from templates using Power Automate.

📊

Status tracking

Move contracts through Draft → In Review → Approved → Signed → Active.

User journey

From contract intake to signature — in nine steps

A real-world CLM workflow across Legal, Procurement, and Approvers. The CLM Agent supports the user at every step and always confirms before irreversible actions.

Legal user

Upload contracts

Legal user uploads new vendor contracts to the enterprise repository.

Agent

Index & enrich

Agent chunks, embeds, and enriches contracts in Azure AI Search.

User

Search a contract

User asks about a specific contract, counterparty, or clause.

Agent

Retrieve clauses

Agent returns exact clause text quoted verbatim, with citations.

Agent

Summarize obligations

Structured brief: obligations, unusual terms, risk score, key dates.

User

Request approval

User asks the agent to route a contract change for approval.

Agent

Trigger workflow

After confirmation, Logic App fires an approval email to Legal.

Approver

Contract approved

Legal approver reviews and clicks Approve. Decision flows back.

Agent

Update status

Agent updates contract state and notifies the requester.

Learning outcomes

Objectives & prerequisites

What you'll learn, and what you need before you start.

🎯 Learning objectives

  • ✓ Build a CLM Agent in Microsoft Foundry with a strong contract-expert persona.
  • ✓ Ground the agent with an enterprise contract repository and force citations.
  • ✓ Connect tools for approvals, doc-gen, and status tracking.
  • ✓ Evaluate the agent for accuracy, groundedness, and safety.
  • ✓ Deploy and share the solution across Web, Teams, and API.

✅ Prerequisites

  • 🧠 Microsoft Foundry access (project on ai.azure.com).
  • ☁️ An Azure subscription or Foundry sandbox.
  • 📄 A handful of sample contracts (MSA / NDA / SOW — real or synthetic).
  • 📚 Basic understanding of what an agent is (model + instructions + tools).
  • 🧰 Optional: VS Code, GitHub Copilot, az CLI.
