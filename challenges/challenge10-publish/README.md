# Challenge 10 — Enterprise Ready

## Scenario

The agent works. Now make it enterprise-ready for rollout to 5,000 employees.

## Goal

Publish the agent with proper governance, security controls, and lifecycle management.

## Instructions

1. **Publish** the agent via Foundry's publish flow.
2. Configure **access controls** — who can use the agent, who can modify it.
3. Set up **deployment slots** (staging vs. production) so you can update safely.
4. Document the **runbook**: how to update, roll back, and escalate issues.
5. Review the **compliance checklist** provided in `coach-guide/compliance-checklist.md`.

## Governance Checklist

- [ ] Agent has a documented owner
- [ ] Data residency requirements met
- [ ] Content Safety policies configured
- [ ] Access control reviewed by security team
- [ ] Monitoring alerts are active
- [ ] Runbook is documented and tested
- [ ] Rollback plan is in place

## Learning Objectives

- Apply enterprise governance to an AI agent
- Understand agent lifecycle management
- Use Foundry's publish and access-control features

## Key Insight

Enterprise readiness is not a feature you add at the end — it's a design decision from day one.
